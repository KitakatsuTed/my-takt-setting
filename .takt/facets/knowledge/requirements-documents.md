# 要件定義ドキュメント知識

## コアドキュメント11種の構成

Webアプリ開発（3〜6名チーム、クライアント協業体制）における標準的な要件定義ドキュメントセット。

### ビジネス要件定義書（BRD）

全ドキュメントの起点。プロジェクトの「なぜ」を定義する。

| セクション | 内容 |
|-----------|------|
| プロジェクトの目的・背景 | なぜこのプロジェクトが必要か |
| ビジネスゴール・KPI | 定量的な成功基準 |
| 対象ユーザー・ステークホルダー | 利害関係者の一覧 |
| スコープ（対象範囲・対象外） | 何をやり、何をやらないか |
| 期待される効果・ROI | ビジネス上の期待値 |
| 前提条件・制約条件 | プロジェクトの前提 |

### 業務フロー図（As-Is / To-Be）

業務プロセスの現状と改善後を可視化する。Draw.ioのMCPサーバーでフローチャートとして作成する。

| セクション | 内容 |
|-----------|------|
| 現状業務フロー（As-Is） | 現在のプロセスを図示 |
| 課題一覧 | As-Isの問題点 |
| 改善後業務フロー（To-Be） | システム導入後の理想 |
| システム化対象範囲 | To-Beのどこがシステム化範囲か |
| 業務ルール | 業務上のルール・制約 |

### 要求一覧

ステークホルダーからの要求を一元管理する。

| カラム | 説明 |
|--------|------|
| 要求ID | REQ-NNN形式 |
| 要求内容 | 要求の具体的な記述 |
| 要求元 | 誰からの要求か |
| 優先度 | 必須/重要/あれば良い |
| 対応方針 | 採用/保留/却下 |
| 関連機能ID | FUNC-NNN |

### 機能一覧

要求をシステム機能に分解する。

| カラム | 説明 |
|--------|------|
| 機能ID | FUNC-NNN形式 |
| 機能名 | 機能の名称 |
| 機能概要 | 機能の説明 |
| 優先度 | 必須/重要/あれば良い |
| 関連要求ID | REQ-NNN |
| 関連画面ID | SCR-NNN |

### 画面一覧

機能を画面に対応付ける。

| カラム | 説明 |
|--------|------|
| 画面ID | SCR-NNN形式 |
| 画面名 | 画面の名称 |
| 画面種別 | 一覧/詳細/入力/ダッシュボード等 |
| 概要・用途 | 画面の目的 |
| 関連機能ID | FUNC-NNN |

### 画面遷移図

画面間の遷移関係を可視化する。Draw.ioのMCPサーバーで状態遷移図として作成する。

### ワイヤーフレーム

主要画面のUI構成を可視化する。Draw.ioのMCPサーバーでUIモックアップとして作成する。

### 非機能要件定義書

IPA非機能要求グレード（6大項目）に準拠する。

| 項目 | 内容 |
|------|------|
| 可用性 | 稼働率目標、RTO/RPO |
| 性能・拡張性 | レスポンスタイム、同時接続数 |
| 運用・保守性 | 監視、ログ、バックアップ |
| 移行性 | データ移行、段階展開 |
| セキュリティ | 認証方式、データ保護 |
| システム環境 | インフラ構成、開発環境 |

### セキュリティ要件定義書

セキュリティに特化した詳細要件。OWASP Top 10を参考にする。

| セクション | 内容 |
|-----------|------|
| 認証・認可方式 | ログイン方式、権限管理 |
| データ暗号化 | 通信暗号化、保存時暗号化 |
| アクセス制御 | ロールベース、リソースベース |
| 監査ログ | 操作ログ、アクセスログ |
| 脆弱性対策 | OWASP Top 10対策 |

### 要件合意書

ステークホルダー間の合意を文書化する。

| セクション | 内容 |
|-----------|------|
| 合意対象の要件範囲 | 何について合意するか |
| 合意者・合意日 | 署名・承認情報 |
| 前提条件 | 合意の前提 |
| 除外事項 | 合意範囲に含まないもの |
| 変更手続き | 合意後の変更プロセス |

### プロジェクトスコープ記述書 + 変更管理台帳

スコープの定義と変更の追跡を行う。

| セクション | 内容 |
|-----------|------|
| プロジェクトスコープ | 作業範囲 |
| プロダクトスコープ | 成果物範囲 |
| 主要な成果物一覧 | 納品物リスト |
| 除外事項 | 対象外の明示 |
| 制約条件・前提条件 | 技術・予算・期間の制約 |

## ドキュメント間の依存関係

| 上流ドキュメント | 下流ドキュメント | 依存の内容 |
|----------------|----------------|-----------|
| BRD | 業務フロー図 | ビジネスゴールが業務フローの改善目標 |
| BRD | 要求一覧 | ビジネス要件から要求を導出 |
| 要求一覧 | 機能一覧 | 要求を機能に分解 |
| 機能一覧 | 画面一覧 | 機能を画面に対応付け |
| 画面一覧 | 画面遷移図 | 画面間の遷移を定義 |
| 画面一覧 | ワイヤーフレーム | 各画面のレイアウト |
| 要求一覧 | 非機能要件定義書 | 非機能に関する要求を詳細化 |
| 非機能要件定義書 | セキュリティ要件定義書 | セキュリティ項目を詳細化 |
| BRD + 機能一覧 | プロジェクトスコープ記述書 | スコープの定義 |
| 全ドキュメント | 要件合意書 | 合意対象の参照 |
| 要件合意書 | 変更管理台帳 | 合意後の変更を管理 |
