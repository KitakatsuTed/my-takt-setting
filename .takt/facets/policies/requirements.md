# 要件定義ポリシー

要件定義ドキュメントの品質と一貫性を維持するための行動規範を定義する。

## 原則

| 原則 | 基準 |
|------|------|
| 自律行動 | 質問しない。不明点は仮定を置いて進める |
| 仮定の明示 | 推測・仮定は「※要確認」「※想定」と明記する |
| 標準準拠 | IPA共通フレーム2013・非機能要求グレードに準拠する |
| ID体系統一 | ドキュメント横断で一貫したID体系を使用する |
| 図表正確性 | 図表はDraw.ioのMCPサーバーで作成し、テキストと整合させる |
| テンプレート完全性 | 空欄を残さない。仮の内容でも埋める |

## ID体系

| ドキュメント | IDプレフィックス | 例 |
|------------|----------------|-----|
| ビジネス要件 | BRD- | BRD-001 |
| 要求 | REQ- | REQ-001 |
| 機能 | FUNC- | FUNC-001 |
| 画面 | SCR- | SCR-001 |
| 非機能要件 | NFR- | NFR-001 |
| セキュリティ要件 | SEC- | SEC-001 |
| 変更要求 | CHG- | CHG-001 |

## ID管理

| 基準 | 判定 |
|------|------|
| IDが一貫した体系に従っていない | REJECT |
| 上流ドキュメントで定義されていないIDを下流で参照 | REJECT |
| 下流ドキュメントで上流IDへの参照が漏れている | 警告 |
| ID体系が全ドキュメントで統一されている | OK |

## 自律行動

| 基準 | 判定 |
|------|------|
| 仮定が「※要確認」なしで記載されている | REJECT |
| 「〜でよろしいですか？」と確認を求めている | REJECT |
| 不明点に仮定を置いて進めている | OK |
| 仮の内容で埋められ「※想定」が付いている | OK |

## ドキュメント品質

| 基準 | 判定 |
|------|------|
| 目的・背景が曖昧 | REJECT |
| ステークホルダーが未定義 | 警告 |
| KPI/成功基準が定量的でない | 警告 |
| スコープの境界（対象外）が未記載 | REJECT |
| 優先度が付いていない要求がある | 警告 |
| セクションが空欄のまま | REJECT |

## 図表品質

| 基準 | 判定 |
|------|------|
| 図がテキスト記述と矛盾している | REJECT |
| 図に記載の要素がテキストに存在しない | 警告 |
| テキストに記載の要素が図に存在しない | 警告 |
| 図のレイアウトが読みにくい | 警告 |
| Draw.ioのMCPサーバーを使わず図を省略している | REJECT |
